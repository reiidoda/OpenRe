# Community Growth and Contributor Activation

## Goal
Turn repository visitors into active contributors by reducing onboarding friction and making contribution paths obvious.

## Contributor journey
1. Visitor discovers repo via search.
2. README communicates clear value and easy first action.
3. Contributor finds right issue lane quickly.
4. CI + templates provide fast feedback.
5. Contributor gets merged and returns.

## Activation tactics
- Keep a visible set of `good first issue` and `help wanted` issues.
- Maintain clear milestone mapping and issue acceptance criteria.
- Ensure quick setup + quick first success path in README.
- Add docs index with direct links to architecture and coding contexts.
- Keep PR review turnaround predictable.

## Maintainer checklist
- Label at least 5 open issues as `good first issue`.
- Label deeper tasks as `help wanted`.
- Keep backlog current and remove stale blockers.
- Acknowledge first-time contributors in release notes.

## Contribution UX quality metrics
- Time to first response on issue.
- Time to first successful CI pass on PR.
- First-time contributor merge rate.
- Returning contributor rate after 30/60 days.

## Governance alignment
- Require quality and safety gates for all contributions.
- Keep contribution standards explicit and repeatable.
- Preserve respectful collaboration via Code of Conduct.
