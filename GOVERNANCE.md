# Governance

## Maintainer model

- Core maintainers own release decisions and policy changes.
- Contributors propose changes through issues and PRs.

## Decision model

- Architectural decisions are recorded as ADRs in `docs/17_adrs`.
- Breaking changes require ADR + migration notes.

## Release policy

- Semantic versioning with `0.x` while interfaces evolve.
- Each minor release includes benchmark snapshots and changelog updates.

- Governance contact: `maintainers@open-agent-workbench.local`
