# Security Policy

## Supported versions

This project is pre-1.0 (`0.x`). Security fixes are applied to the latest minor release.

## Reporting a vulnerability

- Email maintainers via the security contact in `GOVERNANCE.md`.
- Include reproduction steps, impact, and affected version.
- Do not open public issues for active vulnerabilities.

## Security boundaries

- High-risk and destructive actions require approval.
- Browser/computer operations should use allowlists and isolated environments.
- Secrets must be redacted from logs and traces.
