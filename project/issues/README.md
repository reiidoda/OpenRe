# Issue Backlog Pack

This folder contains ready-to-open issue specs grouped by milestone.

## How to use

1. Create a GitHub issue from a file in `project/issues/m*/`.
2. Keep the same title and acceptance criteria.
3. Apply labels noted in each file.
4. Start a branch: `fix/issue-<id>-<slug>`.

## Milestone groups

- `m0`: foundation
- `m1`: single-agent benchmark runner
- `m2`: evals and regression
- `m3`: optimizer loop
- `m4`: safety and approval
- `m5`: multimodal demo and polish
